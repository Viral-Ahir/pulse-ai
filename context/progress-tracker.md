## Session Notes

- `components/ui/*` files are generated by shadcn — do not modify them.
- App palette CSS vars (`--bg-*`, `--text-*`, `--accent-*`, `--border-*`, `--state-*`) are defined in `:root` in `globals.css`.
- Prisma uses multi-file schemas (`prisma.config.ts` → `schema: "prisma/"`); add new models as `prisma/models/*.prisma`. Generator output `app/generated/prisma/` is gitignored — import client via `@/app/generated/prisma/client`.
- Project ID === Liveblocks room ID. The id is generated client-side as `${toSlug(name)}-${6charSuffix}` and passed as `id` in `POST /api/projects`. The API validates `^[a-z0-9-]+$` and returns `409` on collision.
- Server-only data helpers live in `lib/projects.ts` (guarded with `import "server-only"`). Client-side imports of the `ProjectListItem` type still work via `import type`.
- Workspace access checks live in `lib/project-access.ts` (also `import "server-only"`): `getCurrentIdentity()` resolves Clerk userId + primary email, and `getProjectIfAccessible(projectId, identity)` returns the project only when the caller is the owner or a collaborator. Missing project and unauthorized access both return `null` so callers route to a single `AccessDenied` UI.
- The shadcn `Button` in this project wraps `@base-ui/react/button` and does not support `asChild`. To style a link like a button, apply `buttonVariants({ variant })` directly to `next/link`.
- Collaborator enrichment with Clerk: bulk-fetch via `clerkClient().users.getUserList({ emailAddress })`, then build an email-to-user map keyed lowercased. There is no local user table — emails are the source of truth in Postgres (`ProjectCollaborator`) and Clerk data is fetched on demand. The `lib/collaborators.ts` helper wraps the Clerk call in try/catch and falls back to email-only so a Clerk outage doesn't break the list.
- Collaborator API surface: `GET /api/projects/[projectId]/collaborators` (owner or collaborator), `POST` same path (owner-only invite, validates email regex + 409 on duplicate), `DELETE /api/projects/[projectId]/collaborators/[collaboratorId]` (owner-only, verifies the row belongs to the project before deleting).
- Liveblocks node client lives in `lib/liveblocks.ts` (`import "server-only"`) and is exposed via `getLiveblocks()` rather than a top-level singleton — lazy so missing `LIVEBLOCKS_SECRET_KEY` doesn't break `next build` (only request-time evaluation needs the secret). Auth uses access tokens (`prepareSession` + `session.allow(roomId, FULL_ACCESS)`) because access is gated at the auth route via `getProjectIfAccessible`; rooms are created with `defaultAccesses: []` so unauthorized ID tokens can never join. `userInfo` carries `name`, `avatar`, and a deterministic `color` derived from the Clerk user ID through `getCursorColorForUser`.
- Client-side canvas wiring lives in `components/editor/canvas-wrapper.tsx` and `components/editor/canvas.tsx`. The wrapper owns the `LiveblocksProvider` + `RoomProvider` and surfaces `ROOM_CONNECTION_ERROR` codes via `useErrorListener` as a single `<CanvasError>` fallback (no `react-error-boundary` dependency). Inside, `<ClientSideSuspense>` waits on `useLiveblocksFlow({ suspense: true })`, which is why suspense hooks are imported from `@liveblocks/react/suspense`. The room ID is the project ID — the wrapper takes a `roomId` prop and the workspace shell passes `projectId`.
- Shared canvas types live in `types/canvas.ts` (`NODE_COLORS`, `NodeColorId`, `NODE_SHAPES`, `NodeShape`, `NODE_SHAPE_DEFAULT_SIZES`, `SHAPE_DRAG_MIME`, `ShapeDragPayload`, `CanvasNodeData`, `CanvasNode`, `CanvasEdge`). The custom React Flow node/edge type names are `canvasNode` and `canvasEdge` — wire `nodeTypes`/`edgeTypes` against these strings when custom rendering ships. `canvasNode` is rendered by `components/editor/canvas-node.tsx`, which delegates the actual geometry to the shared `ShapeView` in `components/editor/node-shape.tsx`.
- Drag-to-create flow: `components/editor/shape-panel.tsx` writes a `ShapeDragPayload` JSON blob to `dataTransfer` under the `SHAPE_DRAG_MIME` type (`application/x-pulse-shape`). `components/editor/canvas.tsx` wraps its surface in `<ReactFlowProvider>` (required so the drop handler outside `<ReactFlow>` can call `useReactFlow().screenToFlowPosition`), handles `onDragOver`/`onDrop` on a wrapper `<div>`, and dispatches the new node through `onNodesChange([{ type: "add", item }])` so Liveblocks syncs it. Node IDs are `${shape}-${Date.now()}-${counter}`. Drop position is centered on the cursor by subtracting half of the dragged width/height from the converted flow position.
- Shape rendering and drag preview: `components/editor/node-shape.tsx` exports `ShapeView`, a single component that handles all six shapes. Rectangle/pill/circle are rendered as styled `<div>`s; diamond/hexagon/cylinder are rendered as inline `<svg>` paths with `viewBox="0 0 100 100"`, `preserveAspectRatio="none"`, and `vectorEffect="non-scaling-stroke"` so they scale to the node bounds without distorting the stroke. Selection raises the border from `var(--border-default)` (width 1) to `var(--accent-primary)` (width 2). The cylinder is two paths — a closed body + an unfilled arc that draws the front edge of the top ellipse. The shape panel pre-renders one `<ShapeView>` per shape inside a fixed, offscreen (`left-[-9999px] top-[-9999px]`) `aria-hidden` container at each shape's default size, then `setDragImage(preview, width / 2, height / 2)` during `dragstart` uses that DOM element as the cursor-following drag ghost (the browser auto-hides it on drop/cancel). `ShapeView` also accepts `placeholder` (rendered at 0.45 opacity when the label is empty) and `hideLabel` (suppresses the label overlay entirely — used by the canvas node when its inline textarea editor is open).
- Node resize + inline label editing (`components/editor/canvas-node.tsx`): selected nodes show resize handles via `<NodeResizer>` from `@xyflow/react` with `minWidth={80}`/`minHeight={48}` floors and `color="var(--accent-primary)"`; handles are 8×8 squares filled with `bg-base` and ringed by the accent color, and the connecting `lineStyle` border is transparent so only the corner handles are visible. `NodeResizer` dispatches `dimensions`/`position` `NodeChange`s through React Flow's normal pipeline, which `useLiveblocksFlow` syncs automatically. Inline editing uses local `isEditing` state plus `useReactFlow().updateNodeData(id, { label })` per keystroke — `updateNodeData` queues a `setNodes` through React Flow's batch context, which diffs into a `replace` change and fires `onNodesChange`, so Liveblocks captures every change. The double-click handler lives on the outer node wrapper and `stopPropagation`s to prevent the pane's `zoomOnDoubleClick`. The textarea has the React Flow `nodrag nopan` class names plus pointer-event handlers that stop propagation, so dragging/panning never starts while typing; the flex wrapper around the textarea is `pointer-events-none` so any click outside the textarea (but inside the node) passes through and lets the textarea blur naturally to close editing. Deselecting the node also exits edit mode.
- Node color toolbar (`components/editor/node-color-toolbar.tsx`): rendered as a child of `CanvasNodeRenderer` only when `selected` is true. Positioned `absolute bottom-full left-1/2 -translate-x-1/2 mb-3` so the panel floats above the node and clears the top resize handles. Each swatch is a `h-5 w-5 rounded-full` button; the fill color goes on `backgroundColor` and the paired text color is exposed as a `--swatch-glow` CSS custom property so Tailwind arbitrary values (`hover:shadow-[0_0_8px_0_var(--swatch-glow)]`, `border-[color:var(--swatch-glow)]`) drive the hover glow and active state purely from the palette — no per-color class explosion. Color swaps are dispatched via `useReactFlow().updateNodeData(id, { color })`, identical to the label-edit path, so Liveblocks syncs the change for free. To prevent drag/pan during interaction, both the wrapper and each swatch carry `nodrag nopan` plus `stopPropagation` on pointer/mouse/click events.
- Node connection handles (`components/editor/canvas-node.tsx`): four `<Handle>` elements, one per side (`top`/`right`/`bottom`/`left`), all `type="source"`. The canvas runs in `ConnectionMode.Loose`, which lets source handles accept incoming connections too — so users can drag from any side to any side without needing a paired target handle. Handles render hidden (`opacity-0`) and fade in on `group-hover` of the node wrapper. The styling override uses Tailwind `!`-important utilities (`!h-2 !w-2 !rounded-full !border !border-base !bg-copy-primary`) so React Flow's default `.react-flow__handle` rules (1px border, 6×6) don't win — these utilities produce small white dots ringed by `var(--bg-base)`.
- Custom edge renderer (`components/editor/canvas-edge.tsx`): `CanvasEdgeRenderer` is wired to the `canvasEdge` type via `EDGE_TYPES` in `canvas.tsx`, and `DEFAULT_EDGE_OPTIONS = { type: "canvasEdge", markerEnd: { type: MarkerType.ArrowClosed, color: var(--text-primary), 16×16 } }` makes every connection created through `onConnect` use the custom type with the arrow. Path geometry is from `getSmoothStepPath({ borderRadius: 8 })`; the same call returns the midpoint coordinates that drive label positioning — never compute the midpoint manually. The visible stroke + arrowhead are drawn by `<BaseEdge>` with `interactionWidth={24}`, which paints an invisible wider parallel stroke so the edge hits a comfortable 24px hover/click target without thickening the visible line. Hover (`onMouseEnter`/`onMouseLeave` on the wrapper `<g>`), selection, and editing all feed an `isActive` flag — at rest the stroke is `var(--text-faint)` at 1.25px; active jumps to `var(--text-primary)` at 1.75px with a 120ms transition. Edge labels render through `<EdgeLabelRenderer>` (portal that bypasses SVG) at `translate(-50%, -50%) translate(labelX, labelY)` and have three idle states: saved label → bright pill, active+empty → dashed-border faint hint, idle+empty → nothing. Editing replaces whichever pill is showing with an auto-growing input — implemented with an inline-grid sizer (`<span>` + `<input>` both at `col-start-1 row-start-1`) so the input width tracks `max(label, placeholder)`. The input commits per keystroke via `useReactFlow().updateEdgeData(id, { label })`; empty strings are normalized to `undefined`. Exit on `blur` / Enter / Escape. Portal wrapper carries `nopan`; the editing label, the pills, and the input all carry `nodrag nopan` + `stopPropagation` on pointer/mouse/click; `pointerEvents` flips to `"none"` on the portal when the edge is idle and empty so the canvas pane gets the clicks instead.
- Starter templates (`components/editor/starter-templates.ts`, `components/editor/starter-templates-modal.tsx`): `CANVAS_TEMPLATES` defines three preset diagrams as plain `CanvasNode`/`CanvasEdge` arrays — small `makeNode`/`makeEdge`/`buildTemplate` helpers keep authoring readable (just shape + color + `(x, y)` for nodes; source/target/optional label for edges). The modal is a standard `<Dialog>` with a 1- or 2-column card grid; each card uses an SVG-only preview rendered by `computeLayout()` — it derives a uniform scale from the node bounding box to a 280×140 viewport (12px padding), projects every node to pixel coords, and draws edges as straight lines between centers. No React Flow instance is mounted for the preview. The navbar surfaces the modal via the optional `onOpenTemplates` prop (`<LayoutTemplate>` icon + "Templates" label); `WorkspaceShell` owns `templatesOpen` and threads it through `CanvasWrapper` → `Canvas` → `CanvasSurface`, where `<StarterTemplatesModal>` is mounted alongside the floating control bars. Import dispatches `{ type: "remove", id }` for every current node/edge followed by `{ type: "add", item }` for every template node/edge through the existing `onNodesChange`/`onEdgesChange` callbacks so Liveblocks captures both the wipe and the replacement; a `requestAnimationFrame` defers `reactFlow.fitView({ duration: 300, padding: 0.2 })` so the viewport eases to frame the newly rendered template.
- Canvas controls + keyboard shortcuts (`components/editor/canvas-controls.tsx`, `hooks/use-keyboard-shortcuts.ts`): the bottom-left pill bar uses the same dark chrome as the shape panel and exposes two groups split by a thin vertical divider — zoom-out / fit-view / zoom-in (calling `useReactFlow().zoomIn`/`zoomOut`/`fitView` with `{ duration: 150 }` for an eased transition) and undo / redo (Liveblocks `useUndo`/`useRedo`, gated by `useCanUndo`/`useCanRedo` — disabled buttons get `opacity-40` + `cursor-not-allowed` and the hover styles are suppressed via `disabled:hover:*`). The Liveblocks history hooks read the room storage timeline that `useLiveblocksFlow` already writes through, so node/edge mutations are automatically undoable without extra wiring. The minimap was removed from `canvas.tsx`. The `useKeyboardShortcuts` hook is invoked inside `CanvasSurface` (so it's under both `ReactFlowProvider` and `RoomProvider`); it attaches one `keydown` listener to `window` and short-circuits whenever `event.target` is an `<input>`/`<textarea>`/`<select>` or has `isContentEditable` true so typing into node/edge labels never fires shortcuts. Bindings: `Cmd/Ctrl+Z` → undo, `Cmd/Ctrl+Shift+Z` → redo, `Cmd/Ctrl+Y` → redo, `+`/`=` → zoom in, `-`/`_` → zoom out. Each matched branch `preventDefault`s.
